# Specification

## 1. User Story

**As a** user,
**I want to** view comprehensive financial reports and export my transaction data,
**so that I can** analyze my spending habits in detail, back up my data, and use it in other tools like Excel.

---

## 2. Acceptance Criteria

### Reports & Charts
- **AC1: Monthly Report:** Provide a summary of income and expenses by category for a selected month.
- **AC2: Yearly Report:** Provide an annual overview with income/expense trends.
- **AC3: Jar Distribution:** Display a Pie or Donut chart showing the distribution of expenses across different Jars.
- **AC4: Spending Trends:** Display a Line chart showing spending trends over time (daily/weekly/monthly).
- **AC5: Category Breakdown:** Display a Bar chart showing expenses broken down by category.
- **AC6: Comparison:** Allow users to compare financial data between different months or years side-by-side.

### Export Features
- **AC7: Export to Excel (.xlsx):** Allow exporting full transaction history with proper formatting.
- **AC8: Export to CSV:** Allow exporting simple transaction data in CSV format.
- **AC9: Database Backup:** specific for Android - allow exporting the raw SQLite database or a JSON dump.
- **AC10: PDF Report:** Generate a formatted PDF report for monthly or yearly summaries.

### Filters & Options
- **AC11: Date Range:** Users must be able to select a custom date range for reports and exports.
- **AC12: Filter by Jar/Wallet:** Users can filter identifying specific Jars or Wallets to include in the report.
- **AC13: Filter by Category:** Users can filter by specific categories.

---

## 3. User Journey

1. **Navigation:** User navigates to a "Reports" or "Analysis" tab in the app.
2. **Dashboard:** User sees a dashboard of key charts (Spending Trend, Category Breakdown).
3. **Detailed Reports:** User taps on a specific chart to see a detailed report (e.g., Monthly Report).
4. **Filtering:** User applies filters (e.g., change date range to "Last 3 Months", select specific Jar).
5. **Exporting:** User taps an "Export" button, selects the format (Excel, CSV, PDF), and shares or saves the file.

---

## 4. Specification by Example (SBE)

### Scenario 1: Exporting Monthly Transaction History
**Given** the user is on the Reports screen
**And** has selected "October 2023" as the date range
**When** the user taps "Export" and selects "Excel"
**Then** the app generates an `.xlsx` file containing all transactions from Oct 1-31, 2023
**And** the file includes columns for Date, Category, Jar, Amount, and Note.

### Scenario 2: Viewing Spending Trends
**Given** the user has transaction data for the last 6 months
**When** the user views the "Spending Trends" chart
**Then** a line chart is displayed showing total expenses for each of the last 6 months
**And** hovering/tapping a data point shows the exact total for that month.
