# งานของ Issue #98

## เป้าหมาย

ทำให้หน้า `Transaction History` บน Web เปลี่ยนช่วงเวลาได้จริงจากปุ่ม `Change Period` โดยรองรับ preset หลักและ `Custom Range` พร้อมอัปเดตรายการธุรกรรม สรุปยอด และพฤติกรรม endless pagination ให้สอดคล้องกับช่วงเวลาที่เลือก

## ขอบเขตงาน

- ทำให้ปุ่ม `Change Period` ใช้งานได้จริง
- รองรับ `This Month`, `Last 30 Days`, `This Year`, และ `Custom Range`
- แยกสถานะระหว่างค่าที่ผู้ใช้กำลังเลือกใน picker กับค่าที่ถูก apply แล้ว
- รีเซ็ต pagination เมื่อช่วงเวลาที่ใช้กรองเปลี่ยน
- แสดง empty state ให้เหมาะกับกรณีที่มีข้อมูลอยู่แต่ไม่ตรงช่วงเวลาที่เลือก

## เกณฑ์ยอมรับ

- ผู้ใช้เปลี่ยนช่วงเวลาแล้ว summary และรายการธุรกรรมอัปเดตตามช่วงเวลานั้น
- `Custom Range` จะมีผลต่อรายการก็ต่อเมื่อกด `Apply Range`
- ถ้าผู้ใช้กด `Cancel` ค่าเดิมที่ใช้งานอยู่ต้องไม่เปลี่ยน
- endless pagination ยังทำงานต่อได้หลังเปลี่ยนช่วงเวลา
- มี test ครอบพฤติกรรมสำคัญและ build ผ่าน
