# แผนและผลการทำงาน

## Red

- ปรับ test เดิมให้ไม่ผูกกับวันปัจจุบันแบบเปราะ
- เพิ่ม test สำหรับกรณี `Custom Range` ที่ยังไม่กด apply ต้องยังไม่กระทบผลลัพธ์ที่แสดง
- เพิ่ม test สำหรับกรณี `Cancel` ต้องคืนค่าช่วงเวลาที่ใช้งานอยู่ก่อนหน้า

## Green

- แยก state ของช่วงเวลาที่ใช้งานจริง (`active`) ออกจาก state ใน period picker
- ให้ preset ถูก apply ทันทีเหมือนเดิม แต่ `Custom Range` จะ apply เฉพาะตอนกดปุ่ม
- เมื่อกด `Cancel` ให้ปิด picker และคืนค่า draft ให้ตรงกับค่าที่ apply แล้ว
- ปรับ empty state ให้บอกว่า `No matches found` เมื่อมีข้อมูลอยู่แต่ไม่ตรงช่วงเวลาหรือ filter ที่เลือก

## Refactor

- ปรับ fixture ของ test pagination ให้เรียงวันที่สมจริงขึ้นและลด warning จากข้อมูลจำลอง
- รัน test เฉพาะส่วนที่เกี่ยวข้องและ `npm run build` เพื่อยืนยันว่าโค้ดยังผ่าน
